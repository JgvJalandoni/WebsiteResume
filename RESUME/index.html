Geneva Clinic Management System

Project Overview

GitHub Repository: View on GitHub

This project represents my first successful professional deployment, commissioned by Dr. Gene Garrucho Dela Pena to modernize her recording / database system. The objective was to modernize the clinic's record-keeping by transitioning from a manual system to a robust, digital database solution.

Hardware Implementation & Problem Solving

The project involved unique hardware constraints and sustainability goals. Tasked with repurposing legacy hardware, I successfully integrated the clinic's existing UPS, monitor, and peripherals. When the primary workstation (an Intel NUC 10) was found to be non-functional, I pivoted the architecture to utilize a Raspberry Pi as the dedicated server and workstation. I personally managed the hardware restoration, providing custom electrical cabling and infrastructure to ensure a stable, production-ready environment on low-power, high-efficiency hardware.

  • Developed a hyper-optimized Clinic Management System using Python and CustomTkinter, designed for performance on modest hardware like the Raspberry Pi.
  • Implemented a high-performance caching architecture with dirty-flag tracking for real-time dashboard statistics and O(1) lookups.
  • Engineered an offline-first SQLite database layer with comprehensive indexing and connection pooling for maximum reliability.
  • Built a modern, accessibility-focused UI with advanced filtering, patient history tracking, and automated vital signs logging.
  • Integrated secure admin authentication and automated database backup/export functionality for critical medical data protection.
Python CustomTkinter SQLite3 Raspberry Pi Linux Performance Optimization
